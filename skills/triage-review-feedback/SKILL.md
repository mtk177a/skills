# Triage Review Feedback (レビュー指摘整理)

## 目的

- 外部から来たレビュー指摘を整理し、採否と対応方針を判断できる状態にする。

## 使う場面

- GitHub Copilot や他エージェントのレビューを受けたとき
- 人間レビューの指摘を実装に落としたいとき
- 指摘の妥当性や優先順位を整理したいとき

## 入力 (任意)

- レビュー本文
- 対象差分
- 関連仕様や背景
- `review-changes` の結果

## 手順

1. レビュー指摘を要約する。
2. 指摘ごとに採用、保留、却下を判断する。
3. 判断理由を書く。
4. 採用する場合は優先順位と対応方針を決める。
5. 追加確認が必要な指摘は、確認事項として切り出す。
6. 判断材料が不足する場合は、追加確認事項を明示し、別 agent / subagent の利用は必要時だけ提案する。

## 出力フォーマット

- レビュー要約: ...
- 指摘一覧:
  - ...
- 判断:
  - ...
- 判断理由:
  - ...
- 対応方針:
  - ...
- 追加で確認したい点:
  - ...

## 境界

### Always:

- 指摘の内容説明と採否理由を分けて書く
- 出力は日本語で書く
- 採用する指摘には優先順位を付ける
- 単体で実装着手や保留判断に進める粒度まで対応方針を書く
- 別 agent / subagent は既定で使わず、まずこの Skill 単体で採否判断を返す

### Ask first:

- 仕様そのものを変更する判断が必要な場合

### Never:

- 理由なしで採否を決める
