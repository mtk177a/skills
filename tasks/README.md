# tasks/

`tasks/` は、このリポジトリで今やりたいこと・後でやることを一時的にためる作業キューです。
GitHub Issues よりも軽く、AI エージェントがすぐ参照して進められることを優先します。

## 使い方

- 1 件の作業につき 1 ファイルを作る
- ファイル名は内容が分かる短い kebab-case にする
- 先に終わった task は削除する
- 完了した task を履歴として残したい場合は、別途この README の運用を見直す

## 運用方針

- `tasks/` 配下は git で管理する
- `README.md` 以外の task ファイルは、必要な間だけ置く
- `done/` ディレクトリは作らない
- task ファイルには次を短く書く
  - 何をしたいか
  - なぜ必要か
  - 受け入れ条件
  - 関連ファイル

## 命名例

- `record-session-handoff-path.md`
- `chain-stage-naming.md`
- `apm-exports-gap.md`

## このリポジトリでの使いどころ

- 直近のレビュー指摘を分解しておく
- 小さな改善や保留事項を一時保管する
- 後で削除する前提の軽い作業メモにする
